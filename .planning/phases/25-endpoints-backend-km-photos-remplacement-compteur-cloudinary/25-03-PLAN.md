Le routeur (motokey-api.js) appelle INCONDITIONNELLEMENT `let b = await body(req)` (~L554-555) pour tout POST/PUT/PATCH, AVANT tout match de route. `body()` (L404-411) fait `s += chunk` (coercition Buffer→string LOSSY) puis JSON.parse → corromprait un multipart. `req.ctx` est posé APRÈS body() (~L560). Donc toute route multipart DOIT être interceptée AVANT L554 et poser `req.ctx` elle-même.

RelevesKm.enregistrer (supabase.js L385-417) — contrat exact :
```javascript
await SBLayer.RelevesKm.enregistrer(garage_id, moto_id, {
  km,                       // requis, numérique
  type_evenement,           // 'lecture' (défaut) | 'remplacement_compteur'
  acteur_type, acteur_id,   // requis, jamais null (throw sinon)
  note = null, photo_url = null
})
// → { accepted:true, releve:{...} }                       succès
// → { accepted:false, km_tente, km_actuel }               rejet trigger monotone (= 409, PAS une exception)
// → throw Error(...)                                       champs requis manquants
```
NB (supabase.js L965) : convention acteur garage = acteur_type:'garage', acteur_id: ctx.user_id || garage_id.

RBAC (auth/rbac.js) : `requireRole(ctx,'PRO')`, `requireAnyRole(ctx,['CLIENT'])`, `getGarageIdForUser(ctx,SBLayer)`, hiérarchie CLIENT<MECANO<PRO<CONCESSION<ADMIN.
