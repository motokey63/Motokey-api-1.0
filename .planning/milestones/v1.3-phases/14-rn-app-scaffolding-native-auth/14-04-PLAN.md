<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human E2E verification of MAUTH-01/02/03 on device</name>
  <action>Human runs the app (Expo Go / simulator) against the live API and confirms the five checks in how-to-verify below. No code changes; this task is a blocking acceptance gate.</action>
  <what-built>
    The complete native auth experience: branded login / register / mandatory-OTP-verify / forgot-password (OTP reset) screens, encrypted session storage (LargeSecureStore), and proactive token refresh (60s timer + AppState foreground listener). It runs against the live Express API at https://motokey11-production.up.railway.app with no backend changes.
  </what-built>
  <how-to-verify>
    Open the app in Expo Go (scan the QR from Task 1) or a simulator, then check each MAUTH requirement:

    1. MAUTH-01 — Register: tap "Créer un compte", fill nom + a real email you can receive mail at + a password ≥12 chars (with uppercase, digit, special) + matching confirm. Submit. You should land on the 8-digit OTP screen with subtitle "Un code à 8 chiffres a été envoyé à {email}." Enter the code from the email. You should reach the Home screen showing "Bienvenue {email}" and a "Se déconnecter" button. (If your backend requires a separate login after verify, you'll be sent to login with "Email vérifié ! Connectez-vous." — then log in.)
    2. MAUTH-01 — Login: log out, then log back in with the same credentials. You should reach Home with "Bienvenue !".
    3. MAUTH-01 — Password reset: log out, on login tap "Mot de passe oublié ?", enter your email, submit. On the OTP screen enter the emailed code + a new valid password (≥12 chars) + confirm, submit. You should see "Mot de passe mis à jour. Connectez-vous." Log in with the NEW password successfully.
    4. MAUTH-02 — Encrypted persistence: while logged in, fully close the app (swipe it away), then reopen it. You should return DIRECTLY to Home (still logged in) without re-entering credentials — proving the encrypted session was restored from storage.
    5. MAUTH-03 — Proactive refresh on foreground: while logged in, background the app for a while (ideally >5 minutes so the access token nears expiry; if you can wait ~1 hour past expiry even better), then bring it back to the foreground. The app should stay on Home and continue to work with NO "Session expirée" message and NO visible error — the token was refreshed proactively on foreground. (If you left it backgrounded for days/weeks so the refresh token itself expired, expect to be cleanly returned to login with the toast "Session expirée — reconnectez-vous" — that is the correct D-08 hard-expiry behavior.)

    Note any screen where the branding is off (accent should be orange #ff6b00, Inter font, "MotoKey" logo with orange "Key") or any French copy that differs from expectations.
  </how-to-verify>
  <resume-signal>Type "approved" if all five checks pass, or describe which check failed (with what you saw) so a gap-closure plan can be created.</resume-signal>
