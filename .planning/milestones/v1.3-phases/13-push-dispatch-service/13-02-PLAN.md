    1. Look up all tokens for the client (pattern from motokey-api.js ~line 1733, columns per migration 16):
       ```javascript
       const { data: rows, error } = await SBLayer.supabase
         .from('client_device_tokens')
         .select('token')
         .eq('client_id', clientId);
       if (error) {
         console.error('❌ [13] sendPush — lookup tokens échoué pour client', clientId, ':', error.message);
         return { error: error.message }; // never throw
       }
       if (!rows || rows.length === 0) {
         console.log('🔔 [13] sendPush — aucun token actif pour client', clientId);
         return { sent: 0 };
       }
       ```
    2. Fan out — call `sendToToken` once per token. **Per-token idempotency key** so multi-device fan-out is NOT collapsed to a single send by the UNIQUE guard: compose `${idempotencyKey}::${row.token}` for each token. Pass `clientId` through so `push_send_log.client_id` is populated:
       ```javascript
       const results = [];
       for (const row of rows) {
         const perTokenKey = `${idempotencyKey}::${row.token}`;
         const res = await sendToToken(row.token, { ...payload, clientId }, perTokenKey);
         results.push({ token: row.token, res });
       }
       console.log(`🔔 [13] sendPush — fan-out ${results.length} device(s) pour client ${clientId} (key base "${idempotencyKey}")`);
       return { sent: results.length, results };
       ```
       Rationale (state explicitly in a code comment): using the raw `idempotencyKey` unchanged for every token would insert the row on the first token then hit the UNIQUE constraint (23505) for every other token, silently skipping all but one device. Suffixing with the token makes each device dedup independently while keeping the logical event idempotent per device.
    3. Update the export to expose both: `module.exports = { sendToToken, sendPush };`

    Note on `payload.clientId`: Task 1's sendToToken passes `payload.clientId || null` into PushSendLog.insert. sendPush sets it via `{ ...payload, clientId }`; direct sendToToken callers (manual test) simply omit it and it stores null. Do not add a separate parameter.
