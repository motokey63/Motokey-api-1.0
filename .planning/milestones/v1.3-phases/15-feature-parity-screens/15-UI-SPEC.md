# Phase 15 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Platform note:** `mobile-app/` is a React Native (Expo SDK 54, Expo Router) project, not a React/Next.js/Vite web app. The `shadcn_initialization_gate` from the base agent instructions does not apply — there is no `components.json`, no Tailwind, no shadcn registry concept here. This contract instead locks the **existing Phase 14 component kit + brand theme** (already shipped, do not re-derive) and extends it with the minimum new visual vocabulary Phase 15's 5 screens need.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (not applicable — React Native/Expo project) |
| Preset | not applicable |
| Component library | none (React Native core primitives: `FlatList`, `RefreshControl`, `Modal`, `Alert` — plus the existing Phase 14 kit: `Button`, `TextField`, `Toast`, `Logo`, `OtpCodeInput`, all in `mobile-app/components/`. Reuse verbatim; do not fork variants of these.) |
| Icon library | Emoji (Unicode), zero-dependency — exact parity with `MotoKey_Client.html`'s own icon-free precedent. `@expo/vector-icons` is present only as a nested sub-dependency of `expo` and does **not** resolve from `mobile-app`'s own module tree without an explicit `npm install` (confirmed in 15-RESEARCH.md Pitfall 2) — do not import it without adding it to `package.json` first. Default to emoji. |
| Font | System default (RN default, no `fontFamily` override) for all Phase 15 screen text — matches the existing precedent in `app/(auth)/login.tsx` and `app/(app)/home.tsx`, which style text with `fontWeight` only, no `fontFamily`. `Inter_900Black` (loaded in `app/_layout.tsx`) stays reserved exclusively for the `Logo` wordmark component — do not apply it to any new Phase 15 text. |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, badge internal padding |
| sm | 8px | Compact element spacing (list item internal gaps, form field gaps) |
| md | 16px | Default element spacing (card padding, screen horizontal padding) |
| lg | 24px | Section padding, empty-state vertical padding |
| xl | 32px | Layout gaps (between major Fiche Moto sections: historique / plan d'entretien / pneus) |
| 2xl | 48px | Major section breaks (rare — reserve for empty-state top margin) |
| 3xl | 64px | Page-level spacing (not expected to be needed this phase) |

Exceptions: The existing shared `Button` (`paddingVertical: 14`) and `TextField` (`padding: 12`) components predate this contract and are reused unchanged in Phase 15's new forms (add-moto, claim-moto, revoke-motif). Do not "fix" these values and do not introduce new non-4-multiple paddings in Phase-15-authored code — new screens/components must hold to the 4/8/16/24/32/48/64 scale above.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 (21px) — list item secondary text, form helper text, empty-state body copy |
| Label | 13px | 600 (semibold) | 1.3 (17px) — statut badges, meta text (dates, "dernière mise à jour"), form field labels |
| Heading | 18px | 600 (semibold) | 1.25 (22px) — section titles within Fiche Moto ("Historique", "Plan d'entretien", "Pneumatiques"), moto marque+modèle in list/detail |
| Display | 22px | 600 (semibold) | 1.2 (26px) — screen-level headers where needed (e.g. "Mes garages", "Mes réclamations" stack screen titles) |

Exactly 2 weights declared for all Phase-15-authored text: **400 (regular)** and **600 (semibold)**. Bold (700, `Button` primary text) and ExtraBold (800, auth-screen `title` style in `login.tsx`/`register.tsx`) remain locked to their existing Phase 14 components/screens and are reused as-is — do not introduce a third weight into any new Phase 15 screen's own text styles.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `colors.bg` `#f0f2f5` | Screen background for every Phase 15 screen |
| Secondary (30%) | `colors.card` `#ffffff` (list item/card surfaces) + `colors.card2` `#f8f9fb` (nested/secondary surfaces, e.g. expanded devis line-items) | Cards, list rows, form containers, tab bar background |
| Accent (10%) | `colors.acc` `#ff6b00` | Reserved exclusively for: primary CTA buttons (Ajouter ma moto, Réclamer une moto, Valider ce devis, Envoyer la réclamation), active bottom-tab icon/label tint, focused `TextField` border (existing `Button`/`TextField` behavior, unchanged), ghost-link text (e.g. "Voir mes réclamations" secondary nav link) |
| Destructive | `colors.rd` `#dc2626` | Reserved exclusively for: "Refuser" devis button + its confirmation, "Quitter le garage" button + its confirmation modal, error toasts, error-state icon/text |

Accent reserved for: primary action buttons, the active tab bar item, focused input borders, and ghost/secondary-link text — never for status badges, never for decorative backgrounds.

### Status Semantic Colors (data-driven, not part of the 60/30/10 chrome — do not repurpose)

These come from `theme/colors.ts` (locked, Phase 14) and represent the anti-fraud score system — the app's core value. They are semantic data indicators, not decorative accents, and must never be substituted with `colors.acc`.

| Backend value (`couleur_dossier`) | Theme token | Hex | Meaning |
|---|---|---|---|
| `vert` | `colors.gn` | `#16a34a` | Excellent, score ≥ 80 |
| `bleu` | `colors.bl` | `#2563eb` | Bon, score 60–79 |
| `jaune` | `colors.yw` | `#d97706` | Moyen, score 40–59 |
| `rouge` | `colors.rd` | `#dc2626` | Insuffisant, score < 40 |

Note: `rouge`/`#dc2626` is the same hex as the Destructive color — this is intentional (both signal "needs attention"), but they are semantically distinct: `couleur_dossier: 'rouge'` is a read-only status badge, never an interactive/destructive control.

| Empty state heading (Réclamations) | "Aucune réclamation en cours" |
| Empty state body (Réclamations) | "Réclamez une moto déjà enregistrée par un garage pour l'associer à votre compte." — CTA: "Réclamer une moto" |
| Empty state heading (Garages) | "Aucun garage lié" |
| Empty state body (Garages) | "Vos garages partenaires apparaîtront ici une fois une moto liée ou réclamée." (no CTA) |

| Claim form disabled-photo notice (D-02) | "Contactez votre garage pour finaliser la réclamation." (shown after successful claim submission, replacing any photo-upload UI) |
