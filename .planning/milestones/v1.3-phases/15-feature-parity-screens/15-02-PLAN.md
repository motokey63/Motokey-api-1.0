From mobile-app/lib/api.ts — apiFetch returns:
- network failure → `{ ok:false, status:0, data:{ error:{ message:"Serveur inaccessible — l'API est-elle démarrée ?" } } }`
- HTTP error → `{ ok:false, status:401|403|500, data:{...} }`  ← must NOT trigger cache fallback

⚠️ CRITICAL — REAL backend envelope + apiFetch shape (verified against motokey-api.js:366-371
and mobile-app/lib/api.ts:29-35). `apiFetch` sets `data = await res.json()` and returns
`{ ok, status, data }`, so **`res.data` IS the entire envelope**, not the inner payload:
```
res.data === { success:true, data:{ count:2, limite:3, can_add:true, cta_pro:false }, message, timestamp }
```
To reach the actual object/array you MUST go TWO levels deep: `res.data.data.count`,
`res.data.data.reclamations`, `res.data.data.garages`. A one-level unwrap (`res.data.count`,
`res.data.reclamations`, `res.data.garages`) returns `undefined` and silently breaks these
screens at runtime while unit tests on flat fixtures stay green. `parseLimite`,
`parseReclamations`, and `parseGarages` below MUST check `data?.data?.<key>` FIRST, then fall
back to flatter shapes for robustness (see 15-01-PLAN.md's `parseMotosList`/`parseDevisList`
for the identical pattern already applied there).

Backend contracts (from motokey-api.js, confirmed — payload shown here is the INNER `data` key
of the `{success,data,message,timestamp}` envelope, i.e. `res.data.data`):
- GET /client/limite-motos → inner data: `{ count:number, limite:number, can_add:boolean, cta_pro:boolean }`
- POST /client/motos body: `{ marque, modele, plaque, vin, annee, km, mode_acquisition }`
- POST /client/reclamations body: `{ vin_fourni, plaque_fournie, carte_grise_photo_url }`
- GET /client/reclamations → inner data: `{ reclamations: [...] }`
- GET /client/garages → inner data: `{ garages: [...] }`

Jest mock precedent for AsyncStorage: use `jest.mock('@react-native-async-storage/async-storage', () => ({ default: { setItem: jest.fn(), getItem: jest.fn() } }))` OR the community mock at `@react-native-async-storage/async-storage/jest/async-storage-mock`. Verify which resolves under jest-expo before committing (RESEARCH: jest-expo preset). A hand-rolled in-memory jest.fn mock is acceptable and simplest.

Test style: plain Jest describe/it/expect, no component render (see lib/__tests__/session.test.ts).
