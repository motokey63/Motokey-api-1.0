export interface ApiResult<T = any> { ok: boolean; status: number; data: T; }

⚠️ CRITICAL — REAL backend envelope + apiFetch shape (verified against motokey-api.js:366-371
and mobile-app/lib/api.ts:29-35). `apiFetch` sets `data = await res.json()` and returns
`{ ok, status, data }`, so **`res.data` IS the entire envelope**, not the inner payload:
```
res.data === { success:true, data:{ motos:[...], total:N }, message, timestamp }
```
To reach the actual array/object you MUST go TWO levels deep: `res.data.data.motos`.
A one-level unwrap (`res.data.motos`) returns `undefined`, and `res.data.data` returns the
inner *object* (not the array) — both silently break every screen at runtime while unit
tests on flat fixtures stay green. Every list/detail parser below MUST check
`data?.data?.<key>` FIRST, then fall back to flatter shapes for robustness. Do NOT trust the
one-level `data.motos || data.data || []` chain the web client uses — that client unwraps the
envelope earlier; this mobile `apiFetch` does not.

Test style (from mobile-app/lib/__tests__/api.test.ts, session.test.ts): plain Jest,
`describe`/`it`/`expect`, no component rendering, no new testing library. Preset is
`jest-expo` (mobile-app/package.json "jest" key). Run: `cd mobile-app && npx jest lib/__tests__/<file>`.
