<objective>
Wire the missing `brouillon` → `envoye` devis transition that is the real, concrete trigger point for MPUSH-03 ("l'utilisateur reçoit une notification push immédiate quand un nouveau devis est créé pour lui"). Today no code path performs this transition — `POST /devis` always creates `brouillon`, and `GET /devis` filters `brouillon` out for CLIENT — so there is no event to hook `sendPush()` into. This plan adds the "Envoyer au client" backend action (D-01), locks a devis against further edits once sent (D-03), fixes the garage-facing devis list UI so the new button has a correct row to attach to (Pitfall 5), and creates a seed fixture so the whole thing is curl-verifiable without any mobile app involvement.

Purpose: give MPUSH-03 a real, load-bearing event to fire on — this is the backend half of Phase 16, fully independent of the mobile push wiring (Plan 16-02/16-03).
Output: `POST /devis/:id/envoyer` (new), `PUT /devis/:id` statut-guarded, `Devis.envoyer()` (new), `Devis.list()` total-on-read (fixed), `scripts/seed-test-devis-16-uat.js` (new), `app.html` devis list fixed + wired.
</objective>
